I stored Supabase URL and anon key inside a .env.local file instead of hardcoding directly in the source. This prevents credentials from being exposed as well as listing .env.local into .gitignore file to never be commited or pushed.
I then create resepected variables from import.meta.env and before initalizing the client i validate both variables have correctly identified values assigned. If either variables are unidentifed the file throws an error this checks runs when ever any part of the app imports supabaseClient.ts(module load time). without a valid URL or anon key there is no database connection causing any data to be stored or tokesn to be verified.
Passing both URL and key variables to createClient() which returns a configured client instance that is tied to my specifc Supabase project.This istnace is exported to every file that interacts with Supabase to either authorize or CRUD operation.Creating an instnace for every file can cause bottlenecks on read/write performace, one instance to be shared is more optimal
